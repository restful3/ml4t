# 제3장. 백테스팅 — 쉬운 해설판

> 이 글은 Ernest P. Chan의 *"Quantitative Trading"* (2판, 2021) 제3장의 전체 내용을 빠짐없이 담되, 전문 용어와 개념을 일상적인 비유와 풀어쓴 설명으로 재구성한 해설판입니다.

전통적 투자 프로세스와 퀀트 투자 프로세스의 핵심 차이는 **백테스팅(backtesting)** — 전략이 과거에 어떤 성과를 보였을지 시뮬레이션할 수 있다는 것입니다. 아무리 완벽하게 기술된 전략을 발견하더라도 **직접 백테스팅해야** 합니다. 이유는 세 가지입니다:

1. 전략을 완전히 이해하고 정확히 재현했는지 확인
2. 원래 연구에 오류가 없었는지 검증 (과학 연구의 결과 재현과 동일)
3. 전략의 변형을 실험하여 정교화하고 개선

**비유로 설명하면 이렇습니다:**

1장에서 백테스팅을 **메뉴에 올리기 전 시식 테스트** 에 비유했습니다. 이 장은 그 시식 테스트를 **어떤 주방 도구(플랫폼)로, 어떤 재료(데이터)를 사용하여, 어떤 기준(성과 측정)으로** 수행하는지, 그리고 **흔히 저지르는 실수(함정)** 를 다룹니다.

## 일반적인 백테스팅 플랫폼

수만 달러짜리 상용 플랫폼도 있지만, 스타트업에 맞는 무료/저가 도구를 소개합니다.

### Excel

가장 기본적이고 가장 흔한 도구. 핵심 장점은 **WYSIWYG(보이는 대로 얻는다)** — 데이터와 프로그램이 한 곳에 있어 숨겨진 것이 없습니다.

- **선행편향(look-ahead bias)** 이 발생하기 어려움 (날짜와 데이터 열을 쉽게 정렬)
- 백테스팅과 실전 주문 생성을 동일 스프레드시트에서 수행 가능
- 단점: 복잡한 모델 백테스팅에는 부적합 (하지만 단순한 모델이 종종 최고!)

### MATLAB

과거 기관 퀀트의 표준 도구. Python에 밀렸지만 저자는 여전히 **퀀트에게 가장 생산적인 언어** 라고 평가합니다.

- Python보다 쉽고, 빠르며, 공식 고객 지원 제공
- 1,500개 종목을 포함하는 대규모 포트폴리오 전략 테스트에 이상적
- 고급 통계/수학 모듈 내장 (주성분 분석 등)
- Statistics, Econometrics, Financial 등 보조 툴박스 (각 약 $50)
- Trading Toolbox로 실거래도 가능
- 가정용 버전은 Microsoft Office와 비용 비슷

### Python

현재 **사실상의 표준** 백테스팅 언어. numpy, pandas로 MATLAB처럼 배열/시계열 조작 가능. Scikit-learn(머신러닝), plotly(시각화), seaborn(플로팅) 등 풍부한 패키지.

저자가 지적하는 Python의 단점 5가지:

1. **버전 충돌** 이 심함 — 새 패키지 추가가 전체 시스템을 불안정하게 만들 수 있음
2. MATLAB보다 **느림** (학술 연구 Aruoba et al., 2018 인용)
3. **고객 지원 없음** — stackoverflow.com에 의존
4. **IDE가 열등** — VSCode도 MATLAB IDE에 미치지 못함
5. **통계/계량경제학 패키지 부족** — R의 mnormt, copula, fGarch, rugarch, MASS에 미치지 못함

### R

저자가 노스웨스턴 대학교에서 금융 리스크 분석 과정에 사용한 언어. 고전적 통계/계량경제학 분석에 탁월. 학술 통계학자들이 R로 알고리즘을 구현하기 때문. 하지만 머신러닝 구현은 Python/MATLAB만큼 풍부하지 않음.

### QuantConnect

웹 기반 알고리즘 거래 플랫폼. C# 또는 Python으로 연구, 백테스팅, 실거래 가능. 400TB의 금융/대안 데이터 제공. 오픈소스 엔진(LEAN)이 체결, 슬리피지, 마진, 거래비용을 반영한 현실적 백테스트 제공. **백테스팅에서 실거래로 코드 변경 없이 전환 가능.**

### Blueshift

QuantInsti가 제공하는 통합 플랫폼. Python 또는 **비주얼 빌더(프로그래밍 불필요)** 로 전략 개발 가능. 미국, 인도, FX 시장의 분 단위 데이터 무료 제공.

**비유로 설명하면 이렇습니다:**

이 5가지 도구는 식당의 **다른 종류의 오븐** 과 같습니다. Excel은 가정용 오븐(단순하지만 제한적), MATLAB은 전문 셰프용 오븐(강력하지만 비용 있음), Python은 범용 산업용 오븐(강력하지만 설정이 까다로움), QuantConnect/Blueshift는 올인원 스마트 오븐(설정이 쉽지만 커스터마이징 제한)입니다.

**예제 3.1** 에서는 MATLAB, Python, R 각각을 사용하여 Yahoo! Finance에서 주가 데이터를 가져오는 방법을 보여줍니다. 코드는 epchan.com/book에서 다운로드 가능합니다.

## 과거 데이터베이스 찾기와 사용

인터넷에서 놀라울 정도로 많은 무료/저가 데이터베이스를 찾을 수 있습니다. 주요 데이터 소스:

| 소스 | 장점 | 단점 |
|------|------|------|
| **Yahoo! Finance** | 무료, 분할/배당 조정 | 생존편향 있음, 한 번에 1개 종목만 |
| **Sharadar** | 생존편향 없음 | 유료 (저렴) |
| **Algoseek** | 틱 데이터 임대, 태그/식별자 풍부 | 중간 가격 |
| **CSI Data** | 저가, Yahoo/Google의 원천 데이터 | 생존편향 있음 (상장폐지 종목 별도 구매 가능) |
| **CRSP** | 생존편향 없음 | 고가, 월 1회 업데이트 |
| **Interactive Brokers** | 외환 데이터 무료 (계좌 보유 시) | |

### 세 가지 핵심 데이터 이슈

#### 1. 분할/배당 조정 (Split and Dividend Adjustment)

주식 분할이나 배당 지급 시, 분할/배당 이전의 모든 가격을 **승수(multiplier)** 로 조정해야 합니다. 조정하지 않으면 가격 급락이 발생하여 잘못된 거래 신호를 생성합니다.

**예제 3.2** 에서 IGE ETF의 2:1 분할과 여러 차례 배당에 대한 조정 방법을 상세히 보여줍니다. Yahoo! Finance는 이미 조정된 가격을 제공하므로, 가능하면 **이미 조정된 데이터를 사용** 하는 것이 좋습니다.

#### 2. 생존편향 (Survivorship Bias)

2장에서 다루었듯이, 파산/상장폐지/합병된 종목이 빠진 데이터베이스의 문제입니다.

**예제 3.3** 의 충격적인 예시: "저가 주식 매수" 전략을 2001년에 실행한 결과:
- **생존편향 없는 데이터**: 수익률 **-42%** (실제 트레이더가 경험했을 결과)
- **생존편향 있는 데이터**: 수익률 **+388%** (가공의 수익!)

대응법: 스스로 **포인트-인-타임 데이터** 를 수집하거나, 최근 데이터에 집중하여 영향을 줄이거나, Sharadar 같은 저렴한 생존편향 없는 데이터베이스를 사용합니다.

#### 3. 고가/저가 데이터의 노이즈

일간 고가/저가는 시가/종가보다 **훨씬 노이즈가 많습니다.** 기록된 고가 아래에서 매수 지정가를 걸었더라도 체결되지 않았을 수 있습니다 (소량 거래, 다른 거래소 체결, 잘못 보고된 틱 등). 따라서 고가/저가에 의존하는 백테스트는 **덜 신뢰** 할 수 있으며, 거의 항상 수익을 **부풀립니다.**

데이터 검증 방법: 일일 수익률을 계산하고, 평균에서 **4 표준편차 이상** 떨어진 날을 면밀히 검토합니다.

## 성과 측정

퀀트 트레이더가 사용하는 주요 성과 지표는 **샤프 비율, 최대 드로다운, MAR 비율** 입니다. CAGR(연복리성장률)은 분모(한 쪽 자본? 양쪽? 레버리지 적용? 비적용?)에 따라 값이 달라지므로 비교가 어렵습니다.

**MAR 비율** = CAGR / 최대 드로다운. 레버리지에 비교적 독립적입니다.

### 샤프 비율 계산의 미묘한 점들

**달러중립 포트폴리오에서 무위험 수익률을 빼야 하나요?** 답은 **아니오** 입니다. 달러중립 포트폴리오는 자기자금조달(self-financing) — 숏 매도 대금으로 롱을 매수 — 이므로 자금 조달 비용이 거의 없습니다. 증거금 잔액은 무위험 수익률에 가까운 이자를 받으므로 상쇄됩니다.

마찬가지로, 익일 포지션을 보유하지 않는 **일중 거래 전략** 에서도 무위험 수익률을 뺄 필요 없습니다. **자금 조달 비용이 발생하는 경우에만** 무위험 수익률을 차감합니다.

### 샤프 비율의 연환산

기간 $T$(월, 일, 시간)로 계산된 샤프 비율을 연환산하려면:

$$
\text{연환산 샤프 비율} = \sqrt{N_T} \times \text{기간 T의 샤프 비율}
$$

여기서 $N_T$는 1년 안의 기간 $T$의 개수입니다.

- 일간 데이터: $\sqrt{252}$ 를 곱함
- 시간 데이터(NYSE 시장 시간만): $\sqrt{1638}$ 을 곱함 (252일 × 6.5시간)
- **흔한 실수**: $252 \times 24 = 6,048$ 으로 계산 (장중 시간만 사용해야 함!)

**예제 3.4** 에서 IGE의 롱 전용 vs 시장중립 전략의 샤프 비율을 Excel, MATLAB, Python, R로 각각 계산합니다.

**예제 3.5** 에서 최대 드로다운과 최대 드로다운 기간을 Excel, MATLAB, Python, R로 계산합니다. 각 코드는 epchan.com/book에서 다운로드 가능합니다.

## 백테스팅의 흔한 함정

백테스팅이 잘못되면 보통 **실제보다 더 좋은 성과** 를 보여줍니다. 생존편향 외에 두 가지 핵심 함정이 있습니다.

### 1. 선행편향 (Look-Ahead Bias)

거래 시점에 **아직 얻을 수 없는 미래 정보** 를 사용하는 오류입니다.

**예시:**
- "당일 저가의 1% 이내에서 매수" → 장 마감 전에는 당일 저가를 알 수 없음!
- 전체 데이터셋의 회귀 계수로 일일 거래 신호 생성 → 미래 데이터 사용

**대응법:**
- 신호 계산에 항상 **과거 데이터만** 사용 (lag 적용)
- Excel에서는 셀 하이라이팅으로 시각적으로 확인 가능
- MATLAB/Python/R에서는 lag 함수를 신중히 사용

**최종 검증 방법** (저자의 강력한 권고):
1. 전체 데이터로 백테스트 → 포지션 파일 A 저장
2. 최근 $N$일을 제거한 데이터로 백테스트 → 포지션 파일 B 저장
3. A에서 최근 $N$행 제거하여 B와 같은 길이로 맞춤
4. A와 B가 동일한지 확인 — 다르면 선행편향이 있음!

### 2. 데이터 스누핑 편향 (Data-Snooping Bias)

과거 데이터의 **일시적 노이즈에 파라미터를 과적합** 하여 백테스트 성과가 실전보다 부풀려지는 오류입니다.

**핵심 원칙**: 규칙이 많고 파라미터가 많을수록 위험. 저자는 **5개 이하의 파라미터** (진입/청산 임계값, 보유 기간, 이동평균 룩백 기간 포함)를 권장합니다.

또한 파라미터 최적화만이 아니라, 백테스팅을 반복하면서 내리는 **정성적 결정** (개장 시 진입? 마감 시? 대형주? 중형주?)도 데이터 스누핑의 원인이 됩니다. Bailey et al.의 **수축 샤프 비율(Deflated Sharpe Ratio)** 은 백테스트를 얼마나 많이 수정했는지를 반영하여 "진짜" 샤프 비율을 추정합니다.

### 데이터 스누핑 완화 방법

**1. 충분한 표본 크기**

Bailey et al.의 수학적 결과:

| 조건 | 필요한 백테스트 샤프 비율 | 필요한 표본 크기 |
|------|----------------------|---------------|
| 진짜 SR ≥ 0 (95% 확신) | 1 | 681개 데이터 포인트 (일간 2.71년) |
| 진짜 SR ≥ 0 (95% 확신) | 2 | 174개 (0.69년) |
| 진짜 SR ≥ 1 (95% 확신) | 1.5 | 2,739개 (10.87년) |

**2. 표본 외 테스트 (Out-of-Sample Testing)**

데이터를 **훈련 세트** 와 **테스트 세트** 로 분리합니다. 훈련 세트에서 파라미터를 최적화하고, 테스트 세트에서 검증합니다. 테스트 세트의 최소 크기는 위의 수학적 결과에 의해 결정됩니다.

더 엄밀한 방법은 **이동 최적화(moving optimization)** — 파라미터가 변화하는 데이터에 지속적으로 적응하여, 파라미터에 대한 데이터 스누핑을 제거합니다.

**무파라미터 트레이딩 모델**: 저자의 전 동료 포트폴리오 매니저는 "내 모델은 자유 파라미터가 없다"고 자랑했습니다. 이는 파라미터가 전혀 없다는 뜻이 아니라, 모든 파라미터가 이동 룩백 윈도우에서 **동적으로 최적화** 된다는 뜻입니다. 더 지능적인 방법은 7장의 **조건부 파라미터 최적화(CPO)** — 머신러닝으로 매 거래마다 최적 파라미터를 결정합니다.

궁극적인 표본 외 테스트는 **페이퍼 트레이딩** (5장에서 상세 설명)입니다. 공개된 전략을 검증하는 경우, 출판 시점부터 테스트 시점까지의 전체 기간이 진정한 표본 외 기간입니다.

## 예제 3.6: GLD-GDX 페어 트레이딩

이 예제는 **훈련 세트와 테스트 세트 분리** 를 보여줍니다.

**왜 GLD-GDX인가?** GLD는 금 현물 가격을 추종하고, GDX는 금광 주식 바스켓입니다. 직관적으로 이 둘의 가격은 함께 움직여야 합니다(7장에서 공적분 분석으로 증명).

**전략 요약:**

1. **훈련 세트** 에서 GLD와 GDX의 **헤지 비율(hedge ratio)** 을 회귀 분석으로 결정
2. 스프레드 = GLD 가격 - 헤지 비율 × GDX 가격
3. 스프레드가 평균에서 일정 표준편차 이상 벗어나면 진입, 평균 근처로 돌아오면 청산
4. **테스트 세트** 에서 성과 확인

MATLAB, Python, R 코드는 epchan.com/book에서 다운로드 가능합니다. 선행편향 검증 방법도 포함되어 있습니다.

## 전략 정교화

첫 시도에서 훌륭한 성과를 보이지 않을 수 있습니다. 정교화할 때의 원칙: **훈련 세트에서의 개선이 테스트 세트에서도 개선** 을 가져와야 합니다.

흔한 정교화 방법:
- 특정 종목군 제외 (예: 뉴스 영향이 큰 제약 주식, M&A 대상 주식)
- 진입/청산 타이밍이나 빈도 변경
- 종목 유니버스 변경

**핵심**: 정교화는 **펀더멘탈 경제학이나 잘 연구된 시장 현상** 에 근거해야 합니다. 시행착오의 임의적 규칙은 데이터 스누핑을 초래합니다.

### 예제 3.8: 기존 전략의 작은 변형

예제 3.7의 평균회귀 전략에서 **단 하나의 변경** : 종가 대신 **시가에** 포지션을 업데이트합니다. 결과: 거래 비용 전후 모두 샤프 비율이 양수로 변합니다! 이런 작은 변형이 큰 차이를 만듭니다.

## 요약

백테스팅은 전략의 과거 성과를 현실적으로 시뮬레이션하는 것입니다. 미래 성과가 과거를 닮기를 바라지만, 이는 보장되지 않습니다.

이 장에서 다룬 핵심 이슈:

**데이터:**
- 분할/배당 조정
- 일간 고가/저가의 노이즈
- 생존편향

**성과 측정:**
- 연환산 샤프 비율
- 최대 드로다운
- MAR 비율

**함정:**
- 선행편향: 아직 얻을 수 없는 미래 정보 사용
- 데이터 스누핑 편향: 과도한 파라미터 최적화. 충분한 표본, 표본 외 테스트, 모델 단순화로 완화

**거래 비용:** 포함하지 않은 백테스트는 비현실적

**전략 정교화:** 펀더멘탈에 근거한 작은 변형이 효과적

시간과 자원 제약으로 모든 주의사항을 적용할 수 없다면, 일부를 생략하여 전략의 잠재력을 빠르게 파악한 후 나중에 다시 점검해도 됩니다. 가장 철저한 백테스트도 몇 달의 페이퍼 트레이딩이나 실전 거래 후에야 드러나는 문제를 발견할 수 없습니다.

## 핵심 용어 해설

| 용어 | 쉬운 설명 |
|------|----------|
| **백테스팅(backtesting)** | 거래 전략을 과거 데이터로 시뮬레이션하여 성과를 검증하는 과정 |
| **WYSIWYG** | "보이는 대로 얻는다". Excel의 핵심 장점 — 데이터와 로직이 투명 |
| **분할 조정(split adjustment)** | 주식 분할 전 가격에 승수를 적용하여 연속적 수익률을 유지 |
| **배당 조정(dividend adjustment)** | 배당 지급 전 가격에 승수를 적용하여 배당 효과를 반영 |
| **생존편향(survivorship bias)** | 파산/상장폐지 종목이 데이터에서 빠져 성과가 부풀려지는 편향 |
| **선행편향(look-ahead bias)** | 백테스트에서 거래 시점에 얻을 수 없는 미래 데이터를 사용하는 오류 |
| **데이터 스누핑 편향(data-snooping bias)** | 과도한 파라미터 최적화로 과거 노이즈에 모델을 맞추는 오류 |
| **샤프 비율(Sharpe ratio)** | (평균 초과 수익률) / (수익률의 표준편차). 높을수록 좋음 |
| **최대 드로다운(maximum drawdown)** | 전고점 대비 최대 낙폭. 리스크의 핵심 지표 |
| **MAR 비율** | CAGR / 최대 드로다운. 레버리지에 비교적 독립적 |
| **수축 샤프 비율(Deflated Sharpe Ratio)** | 백테스트 수정 횟수를 반영하여 "진짜" 샤프 비율을 추정하는 지표 |
| **훈련 세트 / 테스트 세트** | 데이터를 파라미터 최적화용(훈련)과 검증용(테스트)으로 분리 |
| **이동 최적화(moving optimization)** | 이동 윈도우에서 파라미터를 지속적으로 재최적화하는 방법 |
| **헤지 비율(hedge ratio)** | 페어 트레이딩에서 두 종목 간 적절한 비율 (회귀 분석으로 결정) |
| **포인트-인-타임 데이터** | 각 시점에서 실제로 사용 가능했던 데이터만 포함한 데이터베이스 |
