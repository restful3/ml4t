# 알고리즘 트레이딩 기초 학습 가이드

이 학습 가이드는 알고리즘 트레이딩의 핵심 구성 요소, 데이터 관리, 프로그래밍 언어 선택, 성능 지표 및 포트폴리오 최적화 전략을 포괄적으로 다룹니다. 제공된 자료를 바탕으로 주요 개념을 정리하고, 학습 내용을 확인할 수 있는 연습 문제와 용어 사전을 포함하고 있습니다.

---

## 1. 핵심 개념 정리

### 알고리즘 트레이딩의 구조
알고리즘 트레이딩은 컴퓨터 프로그램이 시장 데이터(과거 또는 실시간)를 입력받아 백테스트 또는 자동 매매를 수행하는 과정입니다. 프로그램은 브로커의 API를 통해 주문을 제출하고 주문 상태 알림을 다시 받습니다. 백테스트와 실제 거래에서 동일한 프로그램을 사용하는 것은 모델의 일관성을 유지하는 가장 좋은 방법입니다.

### 시장 데이터 분석
*   **과거 데이터:** CSI(Daily 데이터), Quandl(데이터 통합 서비스), CRSP(학술적 연구용, 생존 편향 제거 데이터) 등이 주요 공급원입니다.
*   **호가 데이터(BBO):** 종가뿐만 아니라 최선의 매수/매도 호가(Best Bid and Offer)를 사용하는 것이 백테스트 성과 왜곡을 방지하는 데 중요합니다.
*   **옵션 데이터:** 가격 외에도 Greeks, 내재 변동성, 내재 변동성 표면(Implied Volatility Surface) 정보를 포함하며, 거래가 빈번하지 않은 옵션의 경우 종가보다 장 마감 시점의 호가가 더 정확한 지표가 됩니다.
*   **기타 데이터:** 펀더멘털 데이터(Compustat), 기업 실적 전망(IBES, Estimize), 뉴스 및 감성 데이터(Ravenpack 등)가 활용됩니다.

### 프로그래밍 플랫폼 비교
알고리즘 트레이딩 개발에 주로 사용되는 세 가지 스크립트 언어(REPL 언어)의 특징은 다음과 같습니다.

| 특징 | MATLAB | R | Python |
| :--- | :--- | :--- | :--- |
| **사용 용이성** | 최고 (1위) | 보통 (2위) | 보통 (2위) |
| **IDE 완성도** | 최고 (1위) | 낮음 (3위) | 높음 (1위) |
| **속도** | 빠름 (1위) | 가장 느림 (3위) | 보통 (2위) |
| **도구 모음(Toolbox/Package)** | 풍부 (2위) | 매우 풍부 (1위) | 매우 풍부 (1위) |
| **C/C++ 컴파일 지원** | 가능 (1위) | 지원 안 함 (N/A) | 가능 (1위) |
| **브로커 연결성** | 우수 (1위) | 보통 (2위) | 보통 (2위) |
| **가격** | 유료 (2위) | 무료 (1위) | 무료 (1위) |

### 성능 측정 지표 (Performance Metrics)
저자가 강조하는 5가지 핵심 지표는 다음과 같습니다.
1.  **CAGR (복합 연간 성장률):** 레버리지를 일정하게 유지할 때의 복리 수익률입니다.
2.  **Sharpe Ratio (샤프 지수):** 위험 조정 수익률 지표이나, 수익률이 정규 분포를 따른다고 가정하는 한계가 있습니다.
3.  **Maximum Drawdown (최대 낙폭):** 꼬리 위험(Tail Risk)을 측정하는 더 나은 지표입니다.
4.  **Maximum Drawdown Duration (최대 낙폭 지속 기간):** 낙폭이 회복될 때까지 걸리는 시간입니다.
5.  **Calmar Ratio (칼마 지수):** CAGR을 최근 3년간의 최대 낙폭 절댓값으로 나눈 비율입니다. (유사 지표로 전체 기간을 대상으로 하는 MAR 지수가 있습니다.)

## 2. 단답형 연습 문제

**질문 1:** 알고리즘 트레이딩 프로그램이 브로커와 통신하여 주문을 제출하고 상태를 확인하기 위해 사용하는 것은 무엇입니까?
**정답:** API (Application Programming Interface)

**질문 2:** Python 코드를 개발하고 디버깅하는 데 유용한 IDE(통합 개발 환경) 두 가지는 무엇입니까?
**정답:** Microsoft Visual Studio, PyCharm

**질문 3:** 알고리즘 트레이더에게 가장 권장되는 MATLAB 툴박스 세 가지는 무엇입니까?
**정답:** Statistics and Machine Learning, Econometrics, Financial Instruments

**질문 4:** "생존 편향(Survivorship-bias)"이 제거된 데이터를 사용하는 것이 왜 중요합니까?
**정답:** 현재 상장 폐지된 종목의 데이터를 포함하지 않으면 백테스트 결과가 실제보다 좋게 왜곡될 수 있기 때문입니다.

**질문 5:** 외환 거래에서 발생하는 결제 위험(Herstatt risk)을 제거하기 위해 2002년에 설립된 은행은 무엇입니까?
**정답:** CLS 은행 (CLS Bank)

**질문 6:** 상품 선물 거래 계좌는 SIPC 보험의 보호 대상입니까?
**정답:** 아니요 (주식 계좌와 달리 선물 계좌는 보호되지 않습니다.)

**질문 7:** 개인 이름이 아닌 LLC나 LP와 같은 유한 책임 법인 명의로 계좌를 개설할 때의 주요 장점은 무엇입니까?
**정답:** 계좌 자산 이상의 손실(마이너스 에쿼티)이 발생했을 때 개인의 자산을 보호하고 법인 내로 책임을 제한할 수 있습니다.

**질문 8:** 켈리 공식(Kelly Formula)이 제안하는 최적 레버리지의 계산 방식은 무엇입니까?
**정답:** 초과 수익률의 평균 / 초과 수익률의 분산

## 3. 심층 학습을 위한 서술형 문제

**문제 1: 스크립트 언어와 컴파일 언어(C#, C++)의 사용 시점 구분**
연구 및 백테스트 단계에서 MATLAB, Python과 같은 스크립트 언어가 선호되는 이유를 설명하고, 실제 실행 시스템 구축 시 C#이나 C++로 전환하는 이유를 기술하시오.
*   **학습 포인트:** REPL 언어의 연구 생산성, 객체 지향 설계의 복잡성, 실행 지연 시간(Latency) 및 메모리 최적화의 차이점을 중심으로 서술.

**문제 2: 포트폴리오 최적화 전략의 비교**
마코비츠의 '접점 포트폴리오(Tangency Portfolio)'와 '최소 분산 포트폴리오(Minimum Variance Portfolio)', 그리고 '리스크 패리티(Risk Parity)' 전략의 차이점과 각각의 한계점을 설명하시오.
*   **학습 포인트:** 기대 수익률 예측의 어려움, 변동성 기반 자산 배분, 정규 분포 가상의 한계, 시장 폭락 시의 전염(Contagion) 위험 등을 포함.

**문제 3: 데이터 품질이 백테스트 성과에 미치는 영향**
단순 통합 종가(Consolidated closing prices)를 사용하는 것과 거래소별 경매 가격(Auction prices) 또는 BBO(Best Bid and Offer) 데이터를 사용하는 것이 결과에 어떤 차이를 만드는지 기술하시오.
*   **학습 포인트:** 성과 부풀리기 현상 방지, 미체결 거래의 실제 가격 반영, 장 마감 시점의 실제 거래 가능성 등을 중심으로 서술.

## 4. 주요 용어 사전

| 용어 | 정의 및 설명 |
| :--- | :--- |
| **API** | 프로그램 간의 상호작용을 가능하게 하는 인터페이스. 트레이딩에서는 데이터 수신 및 주문 제출에 사용됨. |
| **REPL** | Read-Eval-Print-Loop의 약자. 컴파일 과정 없이 코드를 즉시 실행하고 결과를 확인할 수 있는 대화형 언어 환경. |
| **Survivorship Bias** | 상장 폐지된 기업을 제외하고 현재 생존한 기업들로만 백테스트를 수행하여 성과가 과대평가되는 오류. |
| **BBO (Best Bid and Offer)** | 시장에서 가장 유리한 매수 호가와 매도 호가. 실제 거래 가능 가격을 추정하는 데 중요함. |
| **CAGR** | 기간별 수익률을 복리로 계산한 연평균 성장률. |
| **Calmar Ratio** | CAGR을 최대 낙폭(MDD)으로 나눈 값. 위험 대비 수익을 측정하는 지표. |
| **Kelly Formula** | 자산의 복리 성장률을 최대화하기 위한 최적의 레버리지를 결정하는 수학적 공식. |
| **Herstatt Risk** | 외환 거래에서 한쪽 통화는 지급했으나 상대방이 파산하여 반대 급부 통화를 받지 못하게 되는 결제 위험. |
| **Risk Parity** | 포트폴리오의 각 구성 요소가 전체 위험(변동성)에 동일하게 기여하도록 자산을 배분하는 방식. |
| **Efficient Frontier** | 주어진 위험 수준에서 얻을 수 있는 최대 기대 수익률을 연결한 곡선. |
| **Tangency Portfolio** | 효율적 투자선상에서 무위험 수익률 점과 연결한 직선이 접하는 포트폴리오. 샤프 지수가 극대화됨. |
