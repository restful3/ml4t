# 시계열 분석 — 해설판

> **이 문서는 Ernest P. Chan의 _Machine Trading_ (2016) 제3장 「Time-Series Analysis」 전체를 담은 한국어 해설판입니다.** 원문을 요약하거나 건너뛰지 않고, 모든 문단·수식·코드·표·그림·연습문제·주석을 그대로 살리되, 어려운 대목은 "왜 그런가"를 곁들여 쉬운 한국어로 풀어 설명합니다. 이 장은 AR·ARMA·ARIMA·VAR·VEC 그리고 상태공간 모델(칼만 필터)에 이르기까지, 계량경제학과 신호 처리에서 건너온 선형 시계열 도구들을 실제 트레이딩에 어떻게 쓰는지를 코드와 함께 짚어 갑니다. 저자의 관점은 일관됩니다 — 이론의 내부 원리를 파헤치기보다, 기성 소프트웨어 패키지로 예측을 만들어 돈을 버는 데 초점을 둡니다.

---

## 들어가며 — 왜 시계열 분석인가

시계열 분석이 하려는 일은 한마디로 **시계열의 다음 신호를 예측하는 것** 입니다. 그런데 이것이야말로 트레이더가 매일 하려는 바로 그 일입니다. 이 장은 계량경제학(econometrics)과 신호 처리(signal processing)에서 오랫동안 다듬어져 왔고 계량 투자 커뮤니티에서 널리 받아들여진 도구들에 대한 입문입니다.

> **원문 첫 문장 복원 안내**: 원문의 맨 첫 문장은 스캔·OCR 과정에서 앞부분이 깨져 있습니다("Enext signal in a time series, which is exactly what traders try to do as well."). 문장 앞머리의 드롭캡과 "predict the" 부분이 잘려 나간 것으로, 위 본문은 그 뜻을 온전히 복원한 것입니다.

여러분은 이미 저자의 이전 저서들(Chan, 2009 그리고 2013)에서 일부 시계열 기법이 실제로 쓰이는 것을 보았을지도 모릅니다. 가격 시계열의 **정상성(stationarity)** — 평균·분산 같은 통계적 성질이 시간이 흘러도 변하지 않는 성질 — 이나 **공적분(cointegration)** — 여러 가격이 제각기 떠돌아도 어떤 조합은 안정적으로 묶여 있는 관계 — 을 검정하는 대목이 그렇습니다. 하지만 그런 검정은 ARIMA, VAR, VEC 같은 약어로 불리는 선형 모델링 기법이라는 큰 도구상자의 일부에 지나지 않습니다. 마찬가지로, 거의 모든 기술적 트레이더가 가격의 잡음을 걸러 내려고 이동평균을 써 보았습니다. 하지만 **칼만 필터(Kalman filter)** 같은 고급 신호 처리 필터까지 시도해 본 사람은 몇이나 될까요?

시계열 기법이 가장 빛을 발하는 곳은 펀더멘털 정보나 직관이 부족하거나, 있어도 단기 예측에는 별 쓸모가 없는 시장입니다. 통화(currency)와 비트코인이 딱 여기에 해당합니다. Lyons 교수(2001)는 "… 우리의 교과서적 모델이 설명할 수 있는 월간 환율 변화의 비율은 본질적으로 0이다."라고 썼을 정도입니다. 이 장에서는 통화 수익률 예측에 시계열 기법을 쓰는 예를 몇 가지 다루고, 비트코인 예시는 7장으로 미뤄 둡니다. 그렇다고 시계열 분석이 통화 전용인 것은 아닙니다. 펀더멘털 정보가 넘쳐 나는 주식 시장에서도 기술적 분석이 나름 쓸모가 있듯이, 이 장에서는 시계열 분석을 주식에 적용하는 예도 함께 보여 줍니다.

시계열 분석을 다루는 여느 책과 달리, 이 책은 기법들의 내부 작동 원리를 파고들지 않습니다. 대신 예측을 만들기 위해 **기성 소프트웨어 패키지를 어떻게 쓰느냐** 에만 집중합니다. 대부분의 예제는 MATLAB Econometrics Toolbox로 구현되어 있지만, R 사용자라면 forecast, vars, dlm 패키지에서 비슷한 함수를 찾을 수 있습니다. 즉 이 장의 목표는 "이론가가 되는 것"이 아니라 "도구를 능숙하게 부리는 실무자가 되는 것"입니다.

## AR(p) — 시계열 분석의 첫걸음

### AR(1): 어제가 오늘을 얼마나 끌고 가는가

시계열 분석에서 가장 단순한 모델은 **AR(1)** 입니다. 이름은 거창해 보여도 실은 한 막대(bar)의 가격을 바로 다음 막대의 가격과 연결하는 선형 회귀 모델일 뿐입니다. 오늘 가격이 어제 가격에 회귀한다는 뜻에서 "자기 자신에게 회귀한다(auto-regressive)"는 이름이 붙었습니다.

$$
Y ( t ) - \mu = \phi ( Y ( t - 1 ) - \mu ) + \varepsilon ( t )\tag{3.1}
$$

각 기호를 풀어 보면:

1. **$Y(t)$**: 시간 $t$에서의 가격입니다.
2. **$\mu$**: 시계열의 장기 평균 수준입니다. 가격에서 이 평균을 빼서 "평균으로부터 얼마나 벗어났는가"를 봅니다.
3. **$\phi$ (자기회귀 계수)**: 어제의 이탈이 오늘의 이탈로 얼마나 이어지는지를 나타내는 계수입니다.
4. **$\varepsilon(t)$ (잡음, 혁신)**: 평균이 0인 가우스 잡음입니다. 예측할 수 없는 새 충격이라는 뜻에서 **혁신(innovation)** 이라고도 부릅니다.

이 식이 말하는 바는 단순합니다. 오늘의 가격이 평균에서 벗어난 정도는, 어제의 이탈에 $\phi$를 곱한 만큼에 새 잡음을 더한 값이라는 것입니다.

여기서 중요한 개념이 하나 등장합니다. 시계열의 평균과 분산이 시간에 따라 일정하면 그 시계열을 **약정상(weakly stationary)**<sup>1</sup> 이라고 부릅니다. 그리고 AR(1)은 $|\phi| < 1$ 일 때 약정상이 됩니다(이 증명은 연습문제 3.1로 남깁니다). 약정상 시계열은 동시에 **평균회귀(mean reverting)** 하는 성질도 갖습니다(Chan, 2013). 즉 평균에서 멀어지면 다시 평균 쪽으로 당겨집니다. 반대로 $|\phi| > 1$ 이면 시계열은 한 방향으로 **추세(trend)** 를 그리며 뻗어 나가고, 딱 $\phi = 1$ 이면 어제 값에 순전히 잡음만 더해지는 **랜덤 워크(random walk)** 가 됩니다.

$\phi$ 값을 실제 데이터에서 추정하려면 Econometrics Toolbox의 arima와 estimate 함수를 씁니다.

```matlab
model_ar1=arima(1, 0, 0) % 미지의 모수를 가진 AR(1)을 가정함
model_ar1_estimates=estimate(model_ar1, cl);
```

함수 `arima(p, d, q)` 는 $p=1$, $d=0$ 으로 설정하면 AR(1) 모델로 축소됩니다(더 일반적인 버전은 다음 절에서 다룹니다). estimate 함수는 입력한 가격 시계열을 바탕으로 **최대가능도 추정(maximum likelihood estimation)** — 관측된 데이터가 나올 확률을 가장 크게 만드는 모수를 찾는 방법 — 을 적용해 AR(1)의 모수를 찾아 줄 뿐입니다. 이를 2007년 7월 24일부터 2015년 8월 3일까지의 AUD.USD 1분 중간가격 막대에 적용하면 $\phi = 0.99997$ 이라는 추정값이 나오고, 표준오차는 0.00001 입니다.<sup>2</sup> 결론적으로 AUD.USD는 아주 약하게나마 정상적이긴 하지만, $\phi$가 1에 코앞까지 다가가 있어 사실상 랜덤 워크에 매우 가깝습니다.

여기서 한 가지 실무적 주의가 붙습니다. 저자는 실제 거래 가격이 아니라 **중간가격(midprice)** 으로 검정했는데, 이는 **매수–매도 호가 반등(bid–ask bounce)** 을 줄이기 위해서입니다. 거래 가격은 매수 호가와 매도 호가 사이를 톡톡 튀어 다니는데, 이 튐 자체가 실제로는 거래할 수 없는 허상적 평균회귀를 만들어 냅니다. 중간가격을 쓰면 이 착시를 걷어 낼 수 있습니다.

### AR(p): 더 먼 과거까지 끌어오기

AR(1)을 조금만 일반화하면, 하루 전뿐 아니라 $p$개의 과거 시점까지 끌어오는 **AR(p)** 를 생각할 수 있습니다.

$$
Y ( t ) = \mu + \phi_{1} Y ( t - 1 ) + \phi_{2} Y ( t - 2 ) + \cdot \cdot \cdot + \phi_{p} Y ( t - p ) + \varepsilon ( t ) .\tag{3.2}
$$

보다시피 이것은 결국 하나의 다중 회귀 모델입니다. 시간 $t$의 가격을 종속(반응) 변수로 두고, 시차 $p$ 막대까지 거슬러 올라간 과거 가격들을 독립(예측) 변수로 삼습니다. $p$라는 매개변수가 새로 생겼다는 것은, 데이터에 가장 잘 맞는 최적의 $p$를 우리가 직접 찾을 수 있다는 뜻이기도 합니다.

```matlab
model=arima(pMin, 0, 0) % 미지의 모수를 가진 AR(pMin)을 가정함
```

그렇다면 어떤 $p$가 가장 좋을까요? 많은 통계 모델이 그렇듯 여기서도 **베이즈 정보 기준(BIC, Bayesian information criterion)** 을 씁니다. BIC는 모델의 음의 로그가능도에 비례하되, 거기에 $p$에 비례하는 항을 하나 더 붙여 **복잡성에 벌점** 을 매깁니다. 시차를 무작정 늘리면 데이터에는 잘 맞겠지만 과적합 위험이 커지므로, BIC는 "설명력은 좋으면서도 지나치게 복잡하지는 않은" 지점을 고르게 해 줍니다. 우리 목표는 BIC를 최소화하는 것이고, 이를 무차별 대입(brute-force) 방식으로 전부 훑어 찾습니다.<sup>3</sup>

```matlab
LOGL=zeros(60, 1); % 최대 60개 시차(1시간)까지의 로그가능도
P=zeros(size(LOGL)); % p 값
for p=1:length(P)
model=arima(p, 0, 0);
[∼,∼,logL] = estimate(model, mid(trainset),'print',false);
LOGL(p) = logL;
P(p) = p;
end
```

위 코드에서 `mid` 는 중간가격을 담은 배열입니다. 최적의 $p$를 정하고 나면, 그 $p$에 estimate 함수를 적용해 계수 $\mu, \phi_1, \phi_2, \ldots, \phi_p$를 찾습니다.

```matlab
fit=estimate(model, mid);
```

이 함수들을 2007년 7월 24일부터 2014년 8월 12일까지의 AUD.USD 1분 중간가격 막대에 적용하면 최적값으로 $p = 10$ 이 나오고, 계수는 표 3.1과 같습니다.

이제 이 AR(10) 모델로 2014년 8월 12일부터 2015년 8월 3일까지의 표본외(out-of-sample) 데이터에 대해 예측을 할 수 있습니다.

```matlab
yF=NaN(size(mid));
for t=testset(1):size(mid, 1)
[y, ∼]=forecast(fit, 1, 'Y0', mid(t-pMin+1:t)); % 예측에는 가장 최근 pMin개
yF(t)=y(end);                                    % 데이터 점만 있으면 됨
end
```

여기서 한 가지 시간 규약을 짚어 둘 필요가 있습니다. `yF(t)` 는 시간 $t$까지의 데이터로 만든 예측이므로, 실제로는 **시간 $t+1$의 가격에 대한 예측** 입니다. 다음 막대에 대한 예측이 나오면 이것으로 거래 신호를 만들 수 있습니다. 방법은 간단합니다 — 예측 가격이 현재 가격보다 높으면 사고, 낮으면 팝니다.

예측 가격과 현재 가격의 차이를 구하고,

```matlab
deltaYF=yF-mid;
```

그 부호에 따라 포지션을 매수(+1) 또는 매도(−1)로 잡습니다.

```matlab
pos=zeros(size(mid));
pos(deltaYF > 0)=1;
pos(deltaYF < 0)=-1;
```

이 전략은 표본외 구간에서 **연율화 수익률 158퍼센트** 라는 놀라운 성과를 냈습니다. 자산곡선은 그림 3.1에 있습니다. 다만 이렇게 눈부신 수익을 실제로 실현하려면 중간가격에 체결할 수 있어야 하고, 그러려면 지정가 주문을 관리하는 저지연(low latency) 실행 프로그램이 반드시 필요합니다. 다시 말해 이 수익은 "종이 위에서는" 화려하지만, 실전에서는 체결 인프라가 뒷받침되어야 손에 쥘 수 있습니다.

## ARMA(p, q) — 잡음의 기억까지 끌어들이기

### 시차 10개는 왜 이렇게 많았을까

앞서 AUD.USD에 AR(p)를 적용했을 때 가장 잘 맞는 시차가 10개나 필요했던 것을 기억할 겁니다. 이렇게 시차가 많아지는 것은 AR(p) 모델에서 흔한 일입니다. 모델 구조가 너무 단순하다 보니, 그 단순함을 항의 개수를 늘려서 억지로 보완하려 드는 셈이거든요. 그런데 AR 모델에 **$q$개의 지연된 잡음 항** 을 더 넣어 주면, 필요한 시차 수가 종종 확 줄어듭니다. 이렇게 확장한 것이 바로 **ARMA(p, q)** 모델, 즉 자기회귀 이동평균 과정(auto-regressive moving average process)입니다. 여기서 $q$개의 지연된 잡음 항이 **이동평균(moving average)** 부분을 이룹니다.

$$
\begin{array}{c} { { Y ( t ) = \mu + \phi_{1} Y ( t - 1 ) + \phi_{2} Y ( t - 2 ) + \cdot \cdot \cdot + \phi_{\mathrm{p} } Y ( t - p ) + \varepsilon ( t ) } } \\ { { \nonumber } } \\ { { + ~ \theta_{1} \varepsilon ( t - 1 ) + \cdot \cdot \cdot + \theta_{\mathrm{q} } \varepsilon ( t - q ) } } \end{array} { } { } \nonumber\tag{3.3}
$$

윗줄은 앞서 본 AR(p)와 똑같습니다. 새로 붙은 아랫줄이 핵심인데, 과거의 잡음 $\varepsilon(t-1), \ldots, \varepsilon(t-q)$에 각각 계수 $\theta_1, \ldots, \theta_q$를 곱해 더합니다. 즉 "과거의 예측 오차"까지 기억해 두었다가 현재 값을 설명하는 데 쓰는 것입니다. 과거 가격만 보던 AR 모델에 "과거 충격의 여운"이라는 새로운 정보원을 더한 셈입니다.

식 3.3에서 최적의 $p$와 $q$, 그리고 각 항의 계수를 찾는 절차는 AR(p) 때와 비슷합니다. 다만 이번에는 두 변수($p$와 $q$)에 대해 전수 탐색을 해야 하므로, 이중 for 반복문이 필요합니다.

```matlab
LOGL=-Inf(10, 9); % p는 최대 10, q는 최대 9까지의 로그가능도 (10분)
PQ=zeros(size(LOGL)); % p와 q 값
for p=1:size(PQ, 1)
for q=1:size(PQ, 2)
model=arima(p, 0, q);
[∼,∼,logL] = estimate(model, mid(trainset),'print',false);
LOGL(p, q) = logL;
PQ(p, q) = p+q;
end
end
```

각 $p$와 $q$에 대해 로그가능도를 `LOGL(p, q)` 에 저장하고, $p+q$를 `PQ(p, q)` 에 저장합니다. 후자를 저장하는 이유는 BIC를 최소화할 때 복잡성 벌점 항으로 쓰이기 때문입니다. 그렇다면 이 LOGL과 PQ 행렬에서 BIC를 최소화하는 최적의 $p$, $q$를 어떻게 찾을까요? 두 행렬을 1차원 벡터로 편 다음, aicbic 함수를 적용하고, min 함수로 최솟값을 찾습니다.

```matlab
% 상수를 포함해 p+q+1개의 모수를 가짐
LOGL_vector = reshape(LOGL, size(LOGL, 1)*size(LOGL, 2), 1);
PQ_vector = reshape(PQ, size(LOGL, 1)*size(LOGL, 2), 1);
[∼, bic]=aicbic(LOGL_vector, PQ_vector+1, length(mid(trainset)));
[bicMin, pMin]=min(bic)
```

마지막으로, 이 1차원 BIC 벡터를 다시 2차원 배열로 되돌리되 최솟값에 해당하는 칸만 채워 둡니다. 그래야 그 칸의 행 번호(=$p$)와 열 번호(=$q$)를 눈으로 쉽게 확인할 수 있으니까요.

```matlab
bic(:)=NaN;
bic(pMin)=bicMin;
bic=reshape(bic,size(LOGL))
```

### ARIMA(p, d, q) — 왜 가격을 예측하는가

**ARIMA(p, d, q)** 는 자기회귀 누적 이동평균(autoregressive integrated moving average)을 뜻합니다. 여기서는 금융에서 가장 단순하고 가장 흔한 경우인 $d = 1$ 에만 관심을 두겠습니다. $Y(t)$가 ARIMA($p$, 1, $q$) 모델이라면, 이는 곧 그 차분 $\Delta Y(t)$가 ARMA($p$, $q$) 라는 뜻입니다. 여기서 $\Delta Y(t) = Y(t) - Y(t-1)$ 입니다. $Y(t)$를 가격이 아니라 로그 가격이라고 두면 이해가 훨씬 쉬워집니다. 그러면 로그 수익률을 ARMA($p$, $q$) 로 모델링하는 것이, 로그 가격을 ARIMA($p$, 1, $q$) 로 모델링하는 것과 동등해지기 때문입니다. 다시 말해 $d=1$ 이라는 "차분 한 번"이 바로 가격을 수익률로 바꾸는 연산인 셈이고, 그래서 함수 이름에 통합(integrated)의 I가 들어가 있는 것입니다.

그렇다면 $Y$ 대신 로그 수익률 $\Delta Y$를 ARMA($p$, $q$) 로 모델링하는 편이 더 유리할까요? 가격(또는 로그 가격)을 모델링할 때보다 시차 $p$와 $q$를 더 줄일 수 있다면 유리할 것입니다. 하지만 저자는 안타깝게도 그런 경우를 한 번도 본 적이 없다고 말합니다. 예를 들어 AUD.USD 시계열의 로그를 ARIMA($p$, 1, $q$) 로 모델링하면 $p = 1$, $q = 9$ 가 나옵니다 — 딱히 더 간결해지지 않았습니다.

여기서 헷갈리기 쉬운 지점을 저자가 못박아 둡니다. "로그 가격에 대한 ARIMA($p$, 1, $q$) 모델은 로그 수익률에 대한 ARIMA($p$, 0, $q$) 모델과 동등하다"는 참인 진술을, "로그 가격에 대한 ARMA($p$, $q$) = ARIMA($p$, 0, $q$) 모델은 로그 수익률에 대한 어떤 ARMA($p'$, $q'$) 모델과 동등하다"는 진술과 혼동해서는 안 됩니다. **후자는 거짓** 입니다. $\Delta Y$에 대한 ARMA 모델은 언제나 $Y$에 대한 ARMA 모델로 변환할 수 있습니다. 그러나 그 반대, 즉 $Y$에 대한 ARMA 모델이 항상 $\Delta Y$에 대한 ARMA 모델로 변환되는 것은 아닙니다.

왜 그럴까요? $\Delta Y$에 대한 ARMA 모델은 독립변수로 $\Delta Y$들만 가질 수 있습니다. 반면 $Y$에 대한 ARMA 모델은 $\Delta Y$(두 $Y$의 차이일 뿐이니까요)와 $Y$를 **모두** 독립변수로 가질 수 있습니다. 그러니 $Y$에 대한 모델이 더 유연하고 더 나은 결과를 줍니다. 만약 $\Delta Y$와 $Y$를 모두 독립변수로 갖는 $\Delta Y$ 모델을 원한다면, **VEC(p)** 모델을 써야 합니다. 이는 바로 다음 VAR(p) 절의 끝에서 다룹니다.

## VAR(p) — 여러 종목을 한꺼번에 모델링하기

### 한 종목에서 여러 종목으로

식 3.2의 단순 자기회귀 모델 AR(p)는 $m$개의 다변량 시계열로 손쉽게 일반화됩니다. 이렇게 일반화한 모델을 **벡터 자기회귀 모델(vector autoregressive model)**, 줄여서 VAR(p)라고 합니다. 필요한 작업은 두 가지뿐입니다. 첫째, 자기회귀 계수 $\Phi$를 이제 $m \times m$ 행렬로 해석합니다. 둘째, $m$-벡터인 잡음 $\varepsilon$이 **0이 아닌 횡단면 상관관계(cross-sectional correlations)** 는 갖되 **0의 시계열 상관관계(serial correlations)** 를 갖도록 허용합니다.

이 두 번째 조건이 무슨 뜻인지 풀어 보면 이렇습니다. 서로 다른 시점 $t \neq s$에 대해서는 $\varepsilon_i(t)$가 $\varepsilon_j(s)$와 상관되지 않습니다(시간을 가로질러서는 잡음끼리 무관하다는 뜻). 하지만 **같은 시점** 에서는 $\varepsilon_i(t)$가 $\varepsilon_j(t)$와 상관될 수 있습니다(같은 순간의 종목들끼리는 함께 움직일 수 있다는 뜻). 자기회귀 계수 행렬이 모든 시계열의 현재 가격을 모든 시계열의 지연 가격과 연결해 주므로, VAR 모델은 수익률이 서로 얽혀 움직이는 금융상품 — 예를 들어 같은 산업군에 속한 주식 포트폴리오 — 을 모델링하는 데 특히 잘 맞습니다.

이 절에서는 2007년 1월 3일 기준 S&P 500 지수 안의 컴퓨터 하드웨어 그룹에 초점을 맞춥니다. 이 그룹은 티커 AAPL, EMC, HPQ, NTAP, SNDK로 구성됩니다. 매수–매도 호가 반등에서 비롯되는 허상적 평균회귀 효과를 없애기 위해, 2007년 1월 3일부터 2013년 12월 31일까지 증권가격연구센터(Center for Research of Security Prices, CRSP)가 제공한 장 마감 시점의 중간가격을 사용합니다.

AR(p) 절에서와 마찬가지로, 먼저 최적 지연 $p$를 정해야 합니다. 이를 위해 처음 6년치 데이터를 학습 세트로 씁니다. 필요한 코드에는 사소한 차이만 있습니다.<sup>4</sup>

```matlab
for p=1:length(P)
model=vgxset('n', size(mid, 2), 'nAR', p, 'Constant', true);
% 가법적 오프셋 포함
[model,EstStdErrors,logL,W] = vgxvarx(model,mid(trainset, :));
[NumParam,∼] = vgxcount(model);
LOGL(p) = logL;
P(p) = NumParam;
end
```

반갑게도 BIC를 최소화하는 값으로 $p = 1$ 이 나옵니다(더 단순한 모델이 보통 더 좋습니다). 그리고 이것은 대부분의 산업군에서 나타나는 전형적인 결과이기도 합니다. $p$가 정해지면 모델의 나머지 매개변수는 vgxvarx 함수로 구하는데, 이는 ARIMA 모델의 estimate 함수에 해당합니다. 같은 학습 세트를 써서 얻은 상수 오프셋, 자기회귀 계수, 잡음 항의 공분산을 표 3.3에 정리했습니다. (이 표에서는 표 3.1이나 3.2와 달리, 아래첨자가 시간 지연의 수가 아니라 개별 주식을 가리킵니다.)

2013년의 표본외 데이터에 대해 이 모델로 예측하려면, ARIMA의 forecast 함수와 비슷한 vgxpred 함수를 씁니다.

```matlab
pMin=1;
yF=NaN(size(mid));
for t=testset(1):size(mid, 1)
FY = vgxpred(model,1, [], mid(t-pMin+1:t, :));
yF(t, :)=FY;
end
```

VAR 모델의 선형성에 발맞춰, 우리도 선형 트레이딩 모델을 짤 수 있습니다. 나아가 이를 **섹터 중립(sector-neutral)** 으로 만들 수도 있습니다. 매일 산업군 안 모든 주식의 평균 예측 수익률 $\langle r \rangle$을 구한 뒤, 각 주식의 목표 달러 배분을 "그 주식의 예측 수익률이 산업군 평균에서 얼마나 벗어났는가"에 비례하도록 잡는 것입니다.

각 주식 $i$의 가중치는 다음 식으로 정합니다.

$$
\mathrm { \Sigma }_{w _ { i} } = ( r_{i} - \langle { r } \rangle ) / \sum_{j} | \mathrm{r}_{j} - \langle { r } \rangle | .\tag{3.4}
$$

식을 풀어 보면:

1. **분자 $r_i - \langle r \rangle$**: 주식 $i$의 예측 수익률이 산업군 평균에서 얼마나 벗어났는가. 평균보다 좋게 예측되면 양수(매수), 나쁘게 예측되면 음수(매도)가 됩니다.
2. **분모 $\sum_j |r_j - \langle r \rangle|$**: 모든 종목의 이탈 크기를 절댓값으로 합한 값. 이것으로 나눠 주면 전체 배분의 규모가 정규화됩니다.

이렇게 하면 포트폴리오의 초기 총 시장가치가 항상 \$1이 됩니다. 이 공식이 Chan (2013)의 식 4.1과 비슷해 보일 수 있지만 사실은 다릅니다. 이전 저서의 공식은 전일 수익률을 썼고, 더 중요하게는 평균회귀를 가정했기에 비례상수를 −1로 놓았습니다. 반면 여기서는 예측 수익률을 그대로 쓰고 부호도 뒤집지 않습니다. 각 주식의 포지션(즉 달러 배분)을 계산하는 MATLAB 코드 조각<sup>5</sup> 은 다음과 같습니다.

```matlab
retF=(yF-mid)./mid;
sectorRetF=mean(retF, 2);
pos=zeros(size(retF));
pos=(retF-repmat(sectorRetF, [1 size(retF, 2)]))./repmat
(smartsum(abs(retF-repmat(sectorRetF, [1 size(retF, 2)])), 2),
[1, size(retF, 2)]);
```

이 트레이딩 모델은 **연율 수익률 48퍼센트, 샤프 비율 0.9** 를 냅니다. 자산곡선은 그림 3.3을 보십시오.

### VEC(q) — 가격 대신 가격 변화를 예측하기

우리는 종종 가격 $Y$ 자체보다 가격의 **변화** $\Delta Y$를 예측하고 싶어 합니다. 그런 점에서 VAR 모델을 그대로 쓰는 것은 다소 어색하고, 거기서 나오는 AR 계수도 직관적으로 와닿지 않습니다. 다행히 VAR(p)는 $\Delta Y$를 종속변수로 삼고, 여러 지연된 $\Delta Y$들과 $Y$들을 독립변수로 갖는 모델로 변환할 수 있습니다. 이것을 **VEC(q)**(벡터 오차수정, vector error correction) 모델이라고 하며, 다음과 같이 씁니다.

$$
\Delta Y ( t ) = M + C Y ( t - 1 ) + A_{1} \Delta Y ( t - 1 ) + \cdot \cdot \cdot + A_{k} \Delta Y ( t - k ) + \mathfrak{e} ( t ) .\tag{3.5}
$$

이 식의 핵심은 $Y(t-1)$ 앞에 붙은 $m \times m$ 행렬 $C$입니다. 이것을 **오차수정 행렬(error correction matrix)** 이라고 부릅니다. 어제의 가격 수준 $Y(t-1)$이 오늘의 가격 변화 $\Delta Y(t)$를 어느 방향으로 얼마나 끌어당기는지를 담고 있어서, "수준에서 벗어난 오차를 되돌리는" 힘을 나타냅니다. VAR(p)의 계수를 VEC(q)로 바꾸려면, 먼저 $q = p - 1$ 임에 유의하고 vartovec 함수를 씁니다. 앞서 컴퓨터 하드웨어 주식에 대해 만든 VAR 모델에 이를 적용하면,

```matlab
[model_vec, C]=vartovec(model);
```

## 상태공간 모델 (State Space Models) — 눈에 보이지 않는 변수를 다루기

### 은닉 변수라는 발상

지금까지 살펴본 AR, ARMA, VAR, VEC 모델은 모두 **관측 가능한 변수**(여러 지연 시점의 가격)만으로 미래 값을 예측했습니다. 그런데 계량경제학자들은 여기서 한 걸음 더 나아가, **상태(states)** 라고 불리는 **은닉 변수(hidden variables)** 를 품은 모델 부류도 고안해 냈습니다. 이 은닉 변수는 직접 눈에 보이지 않지만, (관측 잡음의 영향을 받으면서도) 관측되는 변수의 값을 결정합니다. 이런 모델을 **상태공간 모델(SSM, state space model)** 이라 하고, 그 선형적 예가 바로 칼만 필터입니다. 칼만 필터는 Chan (2013)의 3장에서 다뤘고 이 책의 5장에서도 쓰입니다. 비선형 상태공간 모델도 있지만, 이 절에서는 선형 버전만 다룹니다.

상태공간 모델은 은닉 상태 변수(보통 $x$로 표기)가 시간에 따라 어떻게 진화하는지를 명시하는 선형 관계에서 출발합니다.

$$
x ( t ) = A ( t ) * x ( t - 1 ) + B ( t ) * u ( t )\tag{3.6}
$$

각 기호를 풀어 보면:

1. **$x(t)$**: $m$차원의 은닉 상태 벡터. 지금 우리가 관심 있는, 직접 볼 수 없는 값입니다.
2. **$A(t)$**: 상태가 한 시점에서 다음 시점으로 어떻게 넘어가는지를 정하는 $m \times m$ 행렬. 시간에 따라 달라질 수 있지만 관측 가능합니다.
3. **$B(t)$**: 잡음이 상태에 얼마나 섞여 드는지를 정하는 $m \times k$ 행렬.
4. **$u(t)$**: 평균 0, 분산 1, 자기·교차 상관이 모두 0인 $k$차원 가우스 백색잡음.

식 3.6은 흔히 **상태 전이 방정식(state transition equation)** 이라고 부릅니다. 그리고 관측 가능한 변수(측정값이라고도 함)는 또 다른 선형 방정식으로 은닉 변수와 이어집니다.

$$
y ( t ) = C ( t ) * x ( t ) + D ( t ) * \mathfrak{E} ( \mathfrak{t} )\tag{3.7}
$$

여기서:

1. **$y(t)$**: 우리가 실제로 관측하는 $n$-벡터(예: 가격).
2. **$C(t)$**: 은닉 상태를 관측값으로 사상하는 $n \times m$ 행렬.
3. **$D(t)$**: 측정 잡음의 크기를 정하는 $n \times h$ 행렬.
4. **$\varepsilon(t)$**: 평균 0, 분산 1, 자기·교차 상관이 0인 $h$차원 가우스 백색잡음.

식 3.7은 흔히 **측정 방정식(measurement equation)** 이라고 부릅니다. 정리하면, 식 3.6은 "보이지 않는 상태가 어떻게 흘러가는가"를, 식 3.7은 "그 보이지 않는 상태가 우리가 보는 값에 어떻게 드러나는가"를 각각 기술합니다.

### 이동평균을 은닉 변수로 보기

이런 은닉 변수란 대체 무엇이고, 왜 굳이 그 존재를 가정하려는 걸까요? 아주 친숙한 예가 바로 **이동평균** 입니다. 우리는 보통 고정된 개수의 지연 가격으로 이동평균을 계산하니, 겉보기에 이동평균은 관측 가능한 변수처럼 보입니다. 하지만 "그 고정된 지연 개수"라는 것 자체가 인위적인 약속에 불과하다고 볼 수 있습니다. 게다가 단순 이동평균 대신 지수이동평균을 쓰면 안 될 이유도 없지 않나요? 표준적이고 유일한 이동평균이 무엇인지 아무도 합의할 수 없다는 사실이야말로, 이동평균을 **은닉 변수** 로 취급해도 좋다는 신호입니다. 우리는 이 은닉 변수 $x$가 특히 단순한 방식으로 진화하도록 요구함으로써 그것에 어느 정도 구조를 부여할 수 있습니다.

$$
x ( t ) = x ( t - 1 ) + B * u ( t )\tag{3.8}
$$

이 식에서 우리는 $A(t)$를 항등행렬(당연히 시간에 대해 불변)로 가정했습니다. 그래서 상태 $x$는 어제 값에 잡음만 더해져 서서히 흘러갑니다. $B$는 미지이지만 역시 시간에 대해 불변인 행렬로, 이동평균 $x$의 추정 오차 공분산을 정합니다. ($u$ 자체의 공분산 행렬은 항등행렬임을 기억하십시오.) 앞서 $B$가 관측 가능하다고 했지만, 실제로는 학습 데이터에 최대가능도 추정을 적용해 추정할 미지의 모수로 취급할 수 있습니다. (달리 말해 $B$는 칼만 필터 갱신 과정에서 매 시점 갱신되지 않는다는 의미에서만 "관측 가능"합니다.)

시계열의 이동평균(다변량이면 여러 개)이 주어지면, 트레이더는 가격이 추세를 그린다고 가정하고, 시간 $t$의 관측 가격에 대한 최선의 추측을 바로 그 시점의 추정 이동평균으로 삼을 수 있습니다.

$$
y ( t ) = x ( t ) + D * \varepsilon ( \mathrm{t} )\tag{3.9}
$$

여기서 $D$는 역시 MLE로 추정할 미지의 시간 불변 행렬입니다. 즉 "오늘의 가격은 오늘의 이동평균에 약간의 측정 잡음을 더한 값"이라는 아주 소박한 가정입니다.

이 "이동평균" 모델(식 3.8과 3.9)을 실제로 돌려 봅시다. 대상은 앞서 VAR(p) 절에서 다룬 바로 그 컴퓨터 하드웨어 주식 가격 시계열입니다. 은닉 상태 변수의 개수를 컴퓨터 하드웨어 산업군의 주식 수와 같게(총 5개) 잡겠습니다. 이는 전형적인 이동평균 모델의 가정과도 같습니다 — 각 가격 시계열이 저마다 독립적인 이동평균을 갖는다는 것이죠. 나아가 한 이동평균의 상태 잡음은 다른 어떤 것과도 상관되지 않되 각기 다른 분산을 가질 수 있다고 가정합니다. 따라서 $B$는 미지의 모수를 가진 $5 \times 5$ 대각행렬입니다. (미지의 모수는 MATLAB estimate 함수 입력에서 NaN으로 표기합니다.) 비슷하게, 한 주식 가격의 측정 잡음도 다른 주식과 상관되지 않되 각기 다른 분산을 가질 수 있다고 봅니다. 그래서 $D$도 미지의 모수를 가진 $5 \times 5$ 대각행렬입니다. 이 무상관 제약을 풀 수도 있었지만, 그러면 추정할 변수가 훨씬 늘어나 최적화 시간이 폭증하고 과적합 위험도 커집니다. 제약을 거는 것은 답답해 보여도, 실은 모델을 다룰 수 있게 지켜 주는 안전장치입니다.

미지의 상태·측정 잡음 분산($B$와 $D$의 모수)을 추정하기 위해 estimate 함수를 쓰는 코드 조각<sup>6</sup> 은 다음과 같습니다.

```matlab
A=eye(size(y, 2)); % 상태 전이 행렬
B=diag(NaN(size(y, 2), 1))
C=eye(size(y, 2)); % 시간 불변 측정 행렬
D=diag(NaN(size(y, 2), 1))
model=ssm(A, B, C, D);
param0=randn(2*size(B, 1)̂2, 1); % 막대당 50개의 미지 모수
model=estimate(model, y(trainset, :), param0);
```

이 코드는 표 3.5의 값을 만들어 냅니다. 이 경우 $B$와 $D$ 대각 원소의 부호는 의미가 없습니다. 잡음 $u(t)$와 $\varepsilon(t)$가 평균 0을 중심으로 대칭 분포하며 교차상관이 없기 때문입니다. 참고로 가우스 잡음 가정이 더 합당해지도록 가격 대신 로그 가격에 SSM을 적용하는 방안도 고려해 볼 만합니다.

상태 전이 방정식과 측정 방정식이 정해지면, filter 함수로 상태 값과 관측 값 둘 다에 대한 예측을 만들 수 있습니다.

```matlab
[x, logL, output]=filter(model, y);
```

filter 함수 출력의 `x(t)` 변수는 시간 $t$까지 관측된 가격이 주어졌을 때 시간 $t$의 **필터링된 가격**(즉 이동평균)입니다. 이 모델이 만들어 내는 필터링된 가격은 관측 가격과 매우 비슷해서, 보통 차이가 0.1퍼센트도 안 됩니다. 식 3.8과 3.9를 떠올리면, 이는 다음 날 가격에 대한 예측도 결국 오늘 가격과 거의 같아진다는 뜻입니다. 관측 가격이 주어졌을 때 $t$에서의 이 예측 가격은 `output(t).ForecastedObs` 에서 꺼낼 수 있습니다.

```matlab
for t=1:length(output)
yF(t, :)=output(t).ForecastedObs';
end
```

여기서 시간 $t$의 예측 가격을 `yF(t − 1)` 에 배정하는데, 이는 앞에서 쓴 것과 같은 규약입니다. 이 예측 가격들로부터 예측 수익률을 계산합니다.

```matlab
retF=(yF-y)./y;
```

`retF(t)` 는 시간 $t-1$의 관측 가격 $y$가 주어졌을 때 $t-1$에서 $t$까지의 예측 수익률입니다. 이 예측 수익률은 VAR 모델에서와 똑같은 방식으로 섹터 중립 트레이딩 전략을 만드는 데 쓸 수 있습니다. 학습 세트에서의 누적 수익률은 그림 3.4에, 테스트 세트에서의 누적 수익률은 그림 3.5에 나타냈습니다. 학습 데이터로는 그저 상태·측정 잡음의 분산만 추정했을 뿐인데도 **과적합의 정도가 놀라울 만큼 크다** 는 점이 인상적입니다. 인샘플에서는 근사하던 곡선이 아웃오브샘플에서 무너지는 전형적인 모습입니다.

이동평균을 찾는 것만이 칼만 필터로 가격을 예측하는 유일한 길은 아닙니다. 추세 행동을 가정한다면, 최근 가격 추세의 **기울기** 를 찾는 데 칼만 필터를 쓸 수도 있고, 그 기울기가 이어진다고 보면 다음 가격을 예측할 수 있습니다. 이는 독자를 위한 연습문제로 남깁니다(연습문제 3.8).

관측값에 대한 예측을 만드는 것만이 칼만 필터를 트레이딩에 쓰는 유일한 방법도 아닙니다. **은닉 상태 자체의 추정값** 이 유용할 수 있습니다 — 어차피 그것은 이동평균이라고 상정한 값이니까요. 잡음이 낀 상황에서 은닉 변수의 추정값을 찾는 것이 바로 **필터링(filtering)** 의 본래 의미이며, 신호 처리에서 잘 알려진 개념입니다. 칼만 필터 말고도 금융·경제에서 잘 알려진 필터로는 호드릭–프레스콧(Hodrick-Prescott) 필터와 웨이블릿(wavelet) 필터가 있습니다.

### 헤지비율을 은닉 상태로 — EWA와 EWC

칼만 필터의 또 다른 쓰임은 Chan (2013)에서 논의된 바 있는데, 거기서는 공적분하는 두 가격 시계열 사이의 최적 **헤지비율(hedge ratio)** 을 추정하는 데 썼습니다. 예로 든 것은 ETF EWA(하나의 $T \times 1$ 벡터)와 EWC(역시 $T \times 1$ 벡터)의 가격 시계열이며, 이 둘은 다음과 같은 관계로 이어진다고 봅니다.

$$
[ E W C ] = [ E W A , 1 ] * \left[ \begin{array}{c} { { h e d g e r a t i o } } \\ { { o f f s e t } } \end{array} \right] + n o i s e
$$

여기서 발상의 전환이 일어납니다. 두 가격 시계열을 모두 측정값으로 보는 대신, EWC를 측정값 $y$로 두고, 1을 덧붙인 EWA를 식 3.7의 시간 가변 행렬 $C(t)$로 취급합니다. (1을 붙이는 것은 EWA와 EWC 사이 선형 회귀 관계의 상수 오프셋을 허용하기 위해서입니다.) 그리고 헤지비율과 상수 오프셋을 은닉 상태 $x$로 봅니다. 정리하면 다음과 같습니다.

$$
x ( t ) = x ( t - 1 ) + B * u ( t )\tag{3.10}
$$

$$
y ( t ) = C ( t ) * x ( t ) + D * \varepsilon ( \mathrm{t} )\tag{3.11}
$$

각 요소를 풀어 보면:

1. **$x(t)$**: $2 \times 1$ 시간 가변 벡터 [헤지비율, 오프셋]$'$. 이것이 바로 우리가 추정하려는 보이지 않는 값입니다.
2. **$y(t)$**: 스칼라 [EWC($t$)].
3. **$C(t)$**: $1 \times 2$ 시간 가변 행렬 [EWA($t$), 1].

즉 "오늘의 헤지비율과 오프셋"이라는 은닉 상태가 서서히 변해 가고(식 3.10), EWA에 그 헤지비율을 곱한 값이 EWC로 관측된다(식 3.11)는 구조입니다. 헤지비율을 고정된 상수가 아니라 시간에 따라 스스로 갱신되는 은닉 상태로 본다는 점이 이 접근의 묘미입니다. 이 명세에 해당하는 MATLAB 코드 조각은 다음과 같습니다.

```matlab
load('inputData_ETF', 'tday', 'syms', 'cl');
idxA=find(strcmp('EWA', syms));
idxC=find(strcmp('EWC', syms));
y=cl(:, idxC);
C=[cl(:, idxA) ones(size(cl, 1), 1)];
A=eye(2);
B=NaN(2);
C=mat2cell(C, ones(size(cl, 1), 1));
D=NaN;
```

여기서 NaN은 미지의 모수를 가리킵니다. 앞서와 마찬가지로 이 미지 모수들은 2006년 4월 26일부터 2012년 4월 9일까지의 학습 세트에 estimate 함수를 적용해 추정합니다.<sup>7</sup>

```matlab
trainset=1:1250;
model=ssm(A, B, C(trainset, :), D);
```

$B$ 행렬은 표 3.6에 나타냈고, 스칼라 $D$는 −0.08로 추정됩니다. 표 3.5와 달리 여기서는 상태 잡음의 교차상관이 0이라는 제약을 걸지 않았습니다.

$$
\left[ { \begin{array}{r r} { 0 . 0 0 0 5 5 } & { - 0 . 0 1 1 } \\ { - 0 . 0 1 1 } & { 0 . 2 7 } \end{array} \right] .
$$

비슷하게, 측정 잡음 $\varepsilon(t)$의 분산을 임의로 0.001로 놓는 대신, 이번에는 $D^2 = 0.0059$ 로 추정했습니다. 이 추정값들을 써서 filter 함수를 데이터에 적용하면 기울기(그림 3.6)와 오프셋(그림 3.7)의 추정값이 나오는데, 처음에는 Chan (2013)의 그림 3.5·3.6과 꽤 달라 보이지만 결국은 비슷한 값으로 수렴합니다. 이제 이전 저서에서 설명한 것과 같은 트레이딩 전략을 적용할 수 있습니다. 관측된 $y$ 값이 예측값보다 관측치의 예측 표준편차 이상으로 작으면 EWC($y$)를 매수하는 동시에 EWA를 공매도하고, 그 반대의 경우는 정반대로 합니다.

## 요약 (Summary) — 이 장이 남긴 것

시계열 분석은 완전히 새로운 금융상품이나 시장을 마주쳤을 때, 아직 그것에 대한 직관이 전혀 없을 때 **가장 먼저 시도해 볼 기법** 입니다. 이 장에서는 수많은 계량 트레이더의 전략에 스며든, 가장 대중적인 선형 시계열 모델 몇 가지를 훑어보았습니다. AR·ARMA·ARIMA·VAR·VEC 그리고 상태공간 모델까지, 모두 "선형"이라는 공통점을 갖습니다.

하지만 선형이라고 해서 만만한 것은 아닙니다. 추정해야 할 모수가 많아지기 일쑤이고, 그래서 **과적합이 늘 도사리는 위험** 입니다. 특히 상태공간 모델은 자체 동역학을 가진 은닉 변수까지 얹혀 있어 이 위험이 더 큽니다. 이 방법들을 전략 구축에 성공적으로 쓰려면, 미지 모수의 개수를 줄이도록 **현명한 제약을 거는 것** 이 관건입니다. ARMA나 VAR 모델에서는 시차를 1로 제한하는 것이, SSM에서는 잡음의 교차상관을 0으로 두는 것이 널리 쓰이는 제약입니다. 제약을 거는 것을 넘어, 대량의 데이터로 모델을 학습시키는 것이 궁극의 해법입니다. 바로 이 점이 데이터가 풍부한 **일중(intraday) 트레이딩** 에서 이 방법들이 특히 유망하다는 것을 시사합니다.

| 혁신 (innovation) | 예측 불가능한 새 충격을 뜻하는 평균 0의 가우스 잡음 |
| 최대가능도 추정 (MLE) | 관측 데이터가 나올 확률을 가장 크게 만드는 모수를 찾는 방법 |
| 중간가격 (midprice) | 매수·매도 호가의 중간값, 호가 반등의 착시를 줄여 줌 |
| 매수–매도 호가 반등 (bid–ask bounce) | 거래가가 호가 사이를 튀어 생기는 허상적 평균회귀 |
| BIC (베이즈 정보 기준) | 설명력과 복잡성 벌점을 함께 재는 모델 선택 기준, 작을수록 좋음 |
| ARMA(p, q) | AR에 과거 잡음 항($q$개)까지 더한 자기회귀 이동평균 모델 |
| 이동평균 (moving average) | 잡음을 걸러 내려고 과거 값을 평균 낸 것 |
| ARIMA(p, d, q) | 차분($d$번)을 포함한 모델, $d=1$이면 가격을 수익률로 바꿔 다룸 |
| 로그 가격 / 로그 수익률 | 가격에 로그를 취한 값과 그 차분(≈수익률) |
| VAR(p) | 여러 시계열을 행렬 계수로 한꺼번에 다루는 벡터 자기회귀 모델 |
| 횡단면 상관관계 (cross-sectional) | 같은 시점의 서로 다른 종목 잡음 사이 상관 |
| 시계열 상관관계 (serial correlation) | 서로 다른 시점의 잡음 사이 상관(VAR에서는 0으로 가정) |
| 섹터 중립 (sector-neutral) | 산업군 평균 대비 이탈만큼 배분해 시장 방향 노출을 줄인 전략 |
| VEC(q) | $\Delta Y$를 종속변수로, $\Delta Y$·$Y$를 독립변수로 갖는 벡터 오차수정 모델 |
| 오차수정 행렬 (C) | 어제의 가격 수준이 오늘의 변화를 되돌리는 힘을 담은 행렬 |
| Johansen 검정 | 포트폴리오의 공적분 여부를 고윳값으로 판정하는 검정 |
| 상태공간 모델 (SSM) | 관측값 뒤에 숨은 은닉 상태로 시스템을 기술하는 모델 |
| 은닉 변수 (hidden variable) | 직접 관측되지 않지만 관측값을 결정하는 상태(예: 이동평균) |
| 상태 전이 방정식 | 은닉 상태가 시간에 따라 어떻게 진화하는지 기술하는 식(3.6) |
| 측정 방정식 | 은닉 상태가 관측값에 어떻게 드러나는지 기술하는 식(3.7) |
| 칼만 필터 (Kalman filter) | 잡음 속에서 은닉 상태를 추정하는 선형 상태공간 필터 |
| 헤지비율 (hedge ratio) | 두 자산을 짝지어 거래할 때의 상대 비율, 여기선 은닉 상태로 추정 |
| 레짐 변화 (regime change) | 시장 관계 자체가 바뀌어 기존 모델이 무너지는 국면 |
| 과적합 (overfitting) | 학습 데이터엔 잘 맞지만 새 데이터엔 무너지는, 모수 과다의 함정 |
